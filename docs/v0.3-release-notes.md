# v0.3 Release Notes

This page tracks the in-progress `v0.3.0` release. If you want the latest stable build today, use [todoist-assistant-v0.2.7](https://github.com/mtyrolski/todoist-assistant/releases/tag/todoist-assistant-v0.2.7).

## What v0.3 Is

`v0.3.0` is the consolidation release after the `v0.2.x` line. The project grew from a Todoist cache and a few utilities into a local-first dashboard with guided setup, broader automations, optional AI workflows, and a more complete packaging story around the same core data.

## Highlights Across v0.2.x

### Dashboard And Analytics

- Guided first-run setup with token validation and sync progress.
- Cached dashboard bootstrapping so the app can reopen quickly.
- Weekly trend, task lifespan, periodic completion, heatmap, and event timeline plots.
- A more readable active project hierarchy chart.
- Habit tracking summaries and project-level activity cards.
- Better chart polish, spacing, and navigation jump links.

### Automations

- Activity fetching and cache refresh automation.
- Template-driven task creation flows.
- Multiplication and observer automations.
- Gmail ingestion for actionable mail.
- Habit tracking automation.
- LLM breakdown support for expanding tasks into subtasks.

### AI And Chat

- Optional local chat over cached activity.
- Optional OpenAI-backed chat.
- Configurable model/device backend selection.
- Read-only assistant behavior for summaries and explanations.

### Reliability And Data Quality

- Todoist API v1 migration work.
- Better retry and rate-limit handling.
- Activity backfill correctness fixes.
- Current Todoist payload compatibility fixes, including schema drift.
- Expanded tests, CI checks, and data validation coverage.

### Packaging And Distribution

- Windows installer support.
- macOS app and CLI packaging support.
- Docker workflow documentation.
- Improved setup and build documentation.

## Stability Map

- Stable: dashboard, cache/bootstrap flow, core activity analytics, installer and Docker workflows.
- Beta: local chat and other assistant-facing AI surfaces.
- Experimental: lab-style UI surfaces and opt-in dashboard experiments.

## Upgrade Notes

- Existing runtime caches are migrated automatically when needed.
- Migration backups are temporary while the `v0.3.0` release line is still in preparation.
- Runtime logging now defaults to `INFO`, with `TODOIST_LOG_LEVEL=DEBUG` available for verbose traces.
- The latest stable release remains `todoist-assistant-v0.2.7`.
